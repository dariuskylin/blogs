首先，你得安装好git，然后建立你的主目录，里面需包含一些文件及文件夹

基本文件结构 

	/base
		|-- _config.yml  //配置文件
		|-- _layouts     //模板目录
		|-- index.html   //首页文件

其实里面除了_config.yml是必须的，其它的都不是，你完全可以用其它的名字。当然页面都是静态的，可以使用html或是markdown格式的文件。

_config.yml内容

_config.yml有许多的设置，但基本上都可以使用它的默认设置，不用你写，但baseurl是必须指定的，因为它得确认你的项目目录，写法如下： 

	baseurl: /你的目录

这样写是在你使用github的链接地址访问的情况，如果你有域名，那这里就只需要/，而不要带你的目录了

关于_config.yml的其它设置，请看官方

关于基本的操作，可以查看搭建一个免费的，无限流量的Blog

git 发布基本命令

  • 进入根目录
  • 初始化该目录：$ git init
  • 创建一个没有父节点的分支gh-pages：$ git checkout --orphan gh-pages
  • 把内容加入到本地库里：$ git add .（点代表所有文件，你还可以针对某一个文件进行，把.换成路径和文件名就可以了）
  • $ git commit -m "first post"
  • 推送到github：$ git remote add origin https://github.com/你github上的名字/你的项目名字.git（这句只用执行一次）
  • $ git push origin gh-pages

git 克隆命令

  • 进入根目录
  • $ git pull https://github.com/你github上的名字/你的项目名字.git

上传完后，需要等待10分钟左右才能正常访问，但实际上要不了10分钟。
